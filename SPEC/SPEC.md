# PMS (Project Management System) - 요구사항 명세서

## 1. 시스템 개요

요구사항 리스트로부터 기능 리스트, Task, 테스트 시나리오까지의 전체 추적성(Traceability)을 관리하며, AI가 산출물 생성을 보조하는 프로젝트 관리 시스템.

### 1.1 사용자 전제
- 시스템 사용자는 **PMO (Project Management Office)** 조직원
- 동시 사용자 규모: **~10명 내외**

### 1.2 시스템 구성
- **Frontend / Backend 분리 아키텍처**
- **Database**: PostgreSQL
- **배포**: Docker (Docker Compose)
- **다국어 지원** (i18n): 한국어 / English (확장 가능)

### 1.3 인증 및 권한
- **인증 방식**: 이메일 + 비밀번호
- **사용자 역할**: 관리자 (Admin) / 일반 사용자 (User)
  - 관리자: LLM 설정, 사용자 관리, 산출물 템플릿 관리
  - 일반 사용자: 프로젝트 내 모든 기능 사용 (요구사항, 기능, Task, 테스트, 산출물)

### 1.4 AI/LLM 지원
- **지원 프로바이더**: OpenAI (GPT), Anthropic (Claude), Google (Gemini), AWS Bedrock
- 관리자 페이지에서 프로바이더별 API Key, 모델, 프롬프트 설정

## 2. 핵심 데이터 흐름

```
요구사항 입력
  ├── ① 엑셀 Import (.xlsx, 시스템 제공 정제된 템플릿)
  ├── ② 수동 생성 (Web UI)
  ├── ③ 마크다운 업로드 → AI 자동 추출
  └── ④ 요구사항 기술서 Import (고객 원본, 미정제)
        │  엑셀 컬럼: 요구사항구분 / 요청사항 / 상세내용
        │  → AI가 분석/분류/정제 → 요구사항 정의서 초안
        │  → 사용자 검토/수정 → 확정
        │
        ▼
  요구사항 리스트 (확정)
        │  1:N (AI 자동생성 / 수동생성)
        ▼
  기능 리스트 (화면설계서 포함)
        │  1:N (AI 자동생성 / 수동생성)
        ▼
  Task (작업)
        │
        ▼
  테스트 시나리오 → 테스트 케이스 (단위 / 통합)
```

**추적성 체인 (C안 하이브리드)**:
```
요구사항 ──▶ 기능 ──▶ Task
    │          │
    │          └──▶ 테스트 시나리오 ──▶ 테스트 케이스
    │                    ▲
    └────────────────────┘ (직접 연결)
```

## 3. 핵심 모듈

### 3.1 프로젝트 관리
- 다중 프로젝트 생성/관리
- 프로젝트별 대시보드 (진행현황, 일정, 리스크 요약)
- **멤버 관리**: 프로젝트에 사용자 배정/제거 (전체 사용자 목록에서 선택, 접근제한 없음)
- **상태 전이**: 모든 상태 필드는 자유 전이 (어떤 상태에서든 아무 상태로 변경 가능)

### 3.2 요구사항 관리
- 요구사항 CRUD (생성/조회/수정/삭제)
- 요구사항 상태관리 (신규, 검토중, 확정, 변경, 삭제 등)
- 요구사항 우선순위, 분류 관리
- **버전 관리**: 사용자가 의미 있는 시점에 수동으로 버전 저장
- **상세화면 객체 링크**: 연결된 기능/테스트 시나리오를 링크 목록으로 표시 (클릭 → 해당 상세화면 이동)
  - 하위: 기능, 테스트 시나리오(직접 연결)
- **입력 방식**:
  - ① 엑셀 Import (시스템 제공 .xlsx 템플릿 다운로드 → 작성 → 업로드, 정제된 데이터)
    - 템플릿 컬럼: 요구사항 ID(선택, 미입력 시 자동부여) / 분류(필수) / 요구사항명(필수) / 상세설명(필수) / 우선순위(선택, 기본 '중간') / 상태(선택, 기본 '신규') / 비고(선택)
  - ② 수동 생성 (Web UI에서 직접 입력)
  - ③ 마크다운 AI 분석 (.md 파일 업로드 → AI가 요구사항 자동 추출/구조화)
  - ④ 요구사항 기술서 Import (고객 원본 엑셀)
    - 템플릿 컬럼: 요구사항구분(필수) / 요청사항(필수) / 상세내용(필수)
    - **최초 Import**: AI가 미정제 고객 요청을 분석하여 정제된 요구사항 정의서 초안 생성
    - **업데이트 Import**: 기술서 재업로드 시 기존 요구사항과 비교하여 변경분(diff) 분석
      - 신규/변경/삭제/유지 항목을 구분하여 미리보기 표시
      - 사용자가 항목별로 반영 여부 선택 가능
      - 기존 추적성 링크(기능, Task, 테스트 연결) 유지
    - 사용자 검토/수정 → 확정 → 요구사항 리스트 등록/업데이트
- **산출물**: 요구사항 정의서 내보내기

### 3.3 기능 리스트 관리
- 요구사항 <-> 기능 매핑 (1:N)
- AI 자동생성: 요구사항 입력 → 기능 리스트 초안 자동 생성
- 수동생성 + AI 보조 (작성 중 제안)
- 화면설계서 첨부/관리 (기능 단위)
- **버전 관리**: 사용자가 의미 있는 시점에 수동으로 버전 저장
- **상세화면 객체 링크**: 연결된 상위/하위 항목을 링크 목록으로 표시
  - 상위: 요구사항
  - 하위: Task, 테스트 시나리오
- **산출물**: 기능 리스트, 화면설계서

### 3.4 Task 관리
- 기능 <-> Task 매핑 (1:N)
- AI 자동생성: 기능 리스트 → Task 자동 분해
- 진척율 표시 (%)
- 일정 관리 (시작일, 종료일, 마일스톤)
- 이슈 & 리스크 첨부
- 담당자 배정
- **목록 뷰**: 테이블 뷰 (향후 Gantt 차트 뷰 고려)
- **버전 관리**: 사용자가 의미 있는 시점에 수동으로 버전 저장
- **상세화면 객체 링크**: 연결된 상위 항목을 링크 목록으로 표시
  - 상위: 기능
- **산출물**: WBS 내보내기

### 3.5 테스트 관리
- 요구사항/기능 <-> 테스트 시나리오 매핑
- AI 자동생성: 요구사항 + 기능 기반 테스트 시나리오 초안
- 테스트 케이스 관리 (단위 테스트 / 통합 테스트 구분)
- 테스트 수행 결과 기록
- **버전 관리**: 사용자가 의미 있는 시점에 수동으로 버전 저장
- **상세화면 객체 링크**: 연결된 상위/하위 항목을 링크 목록으로 표시
  - 상위: 요구사항, 기능
  - 하위: 테스트 케이스
- **산출물**: 테스트 계획서, 테스트 케이스 문서 내보내기

### 3.6 버전 관리 (Version Control)
- **대상**: 요구사항 정의서, 기능 리스트, Task(WBS), 테스트 시나리오, 테스트 결과서
- 사용자가 **의미 있는 시점에 수동으로** "버전 저장" 이벤트를 발생
- **스냅샷 단위**: 산출물 전체 (예: 요구사항 버전 저장 = 해당 프로젝트 요구사항 전체 목록 스냅샷)
- **버전 생성 시**: 버전명(v1.0, v1.1 등) + 변경 사유 입력 → 현재 상태 전체 스냅샷 저장
- **버전 목록**: 산출물별 버전 타임라인 조회
- **버전 비교**: 두 버전 선택 → diff 뷰 (변경/추가/삭제 항목 하이라이트)
- **버전 복원**: 특정 버전으로 롤백 가능

### 3.7 추적성 관리

#### 링크 구조: 하이브리드 (계층 기본 + 부분 자유)

기본 흐름은 계층 구조를 따르되, 요구사항 ↔ 테스트 시나리오는 직접 연결 허용.

```
요구사항 ──1:N──▶ 기능 ──1:N──▶ Task
    │               │
    │               └──1:N──▶ 테스트 시나리오 ──1:N──▶ 테스트 케이스
    │                               ▲
    └───────────1:N─────────────────┘ (직접 연결 허용)
```

**허용되는 링크:**

| 출발 | 도착 | 관계 | 비고 |
|------|------|------|------|
| 요구사항 | 기능 | 1:N | 계층 (기본) |
| 기능 | Task | 1:N | 계층 (기본) |
| 기능 | 테스트 시나리오 | 1:N | 계층 (기본) |
| 테스트 시나리오 | 테스트 케이스 | 1:N | 계층 (기본) |
| 요구사항 | 테스트 시나리오 | 1:N | **직접 연결** (테스트 추적성) |

**허용되지 않는 링크:**
- 요구사항 → Task 직접 연결 (반드시 기능을 경유)
- 요구사항 → 테스트 케이스 직접 연결 (시나리오를 경유)
- Task → 테스트 시나리오 직접 연결

#### 객체 링크 (상세화면)

각 항목의 상세화면에서 연결된 상위/하위 항목을 양방향 링크 목록으로 표시.
- 클릭 시 해당 항목 상세화면으로 이동
- 연결 항목의 상태/진척율 등 주요 정보를 인라인 표시
- 연결/해제 가능
- **원본 요구사항 항상 표시**: 어떤 상세화면이든 계층을 거슬러 올라가 해당 항목의 원본 요구사항을 항상 표시 (읽기 전용)

**상세화면별 표시 링크:**

| 상세화면 | 원본 요구사항 | 직접 링크 (상위) | 직접 링크 (하위) |
|---------|-------------|----------------|----------------|
| 요구사항 | 자기 자신 | - | 기능, 테스트 시나리오(직접) |
| 기능 | 상위 요구사항 (자동 표시) | 요구사항 | Task, 테스트 시나리오 |
| Task | 기능의 상위 요구사항 (자동 표시) | 기능 | - |
| 테스트 시나리오 | 직접 연결된 요구사항 또는 기능의 상위 요구사항 (자동 표시) | 요구사항(직접), 기능 | 테스트 케이스 |
| 테스트 케이스 | 시나리오를 통해 추적된 요구사항 (자동 표시) | 테스트 시나리오 | - |

**예시 - Task 상세화면 (단일 스크롤, 섹션 구분):**
```
┌─ T-001 회원가입 UI 개발 ─────────────────────┐
│                                               │
│  ┌─ 기본 정보 ─────────────────────────────┐ │
│  │ ...                                      │ │
│  └──────────────────────────────────────────┘ │
│                                               │
│  ┌─ 연결 항목 ──────────────────────────────┐ │
│  │                                          │ │
│  │  📋 원본 요구사항                         │ │
│  │  └── REQ-001 SSO 로그인     확정 →이동   │ │
│  │                                          │ │
│  │  🔧 상위 기능                             │ │
│  │  └── F-001 회원가입 폼      확정 →이동   │ │
│  │                                          │ │
│  └──────────────────────────────────────────┘ │
│                                               │
│  ┌─ 이슈/리스크 ────────────────────────────┐ │
│  │ ...                                      │ │
│  └──────────────────────────────────────────┘ │
│                                               │
│  ┌─ 버전 이력 ──────────────────────────────┐ │
│  │ ...                                      │ │
```

## 4. AI 기능

| 기능 | AI 역할 | 비고 |
|------|---------|------|
| 요구사항 기술서 → 요구사항 정의서 | 분석/분류/정제 | 고객 원본 엑셀 → 정제된 요구사항 초안 |
| 마크다운 → 요구사항 | 자동 추출/구조화 | .md 파일 분석 후 요구사항 항목 생성 |
| 요구사항 → 기능 리스트 | 자동 생성 | 초안 생성 후 사용자 확인/수정 |
| 기능 → Task 분해 | 자동 생성 | 일정/공수 추정 포함 가능 |
| 요구사항+기능 → 테스트 시나리오 | 자동 생성 | 단위/통합 구분하여 생성 |
| 수동 작성 시 | AI 보조 | 작성 중 제안, 자동완성, 누락 항목 체크 |

**모든 AI 생성물은 "초안" → 사용자 검토/수정 → "확정" 워크플로우**

**지원 LLM 프로바이더**: OpenAI (GPT), Anthropic (Claude), Google (Gemini), AWS Bedrock

### 4.1 AI 에러 처리 정책

| 상황 | 처리 방식 |
|------|----------|
| **LLM API 호출 실패** (타임아웃, 서버 오류, 키 만료 등) | 에러 메시지 표시, 사용자가 수동 재시도 |
| **응답 파싱 불가** (JSON 깨짐, 형식 불일치) | 원본 텍스트 그대로 표시, 사용자가 수동 편집 |
| **결과 품질 불량** (엉뚱한 결과) | 미리보기 단계에서 사용자가 판단/필터링 (시스템 자동 판단 안 함) |
| **토큰 한도 초과** (입력 데이터 과다) | "데이터가 너무 많습니다. 범위를 줄여주세요" 안내 메시지 |
| **LLM 미설정 상태** | AI 관련 버튼 비활성화 + 툴팁: "관리자에게 LLM 설정을 요청하세요" |

## 5. 엑셀 Import 템플릿 정의

### 5.1 요구사항 정의서 템플릿 (① 정제된 양식)

시스템이 제공하는 표준 템플릿. 사용자가 다운로드 → 작성 → 업로드.

| 컬럼명 | 필수 | 기본값 | 설명 |
|--------|------|--------|------|

### 5.2 요구사항 기술서 템플릿 (④ 고객 원본, 미정제)

고객이 요구사항을 정제하지 않고 작성하는 양식. AI가 분석/정제.

| 컬럼명 | 필수 | 설명 |
|--------|------|------|
| 요구사항구분 | 필수 | 요청사항을 분류하는 구분 (인증, 화면, 데이터 등) |

| 상세내용 | 필수 | 요청사항의 상세 내용 (자유 형식) |

**샘플 데이터:**

| 요구사항구분 | 요청사항 | 상세내용 |
|------------|---------|---------|
| 인증 | 로그인 | SSO 연동으로 로그인해야 함. 사내 AD 계정 사용 원함 |
| 인증 | 비밀번호 | 비밀번호 분실 시 재설정 가능해야 함 |

| 데이터 | 엑셀 출력 | 모든 데이터는 엑셀로 다운받을 수 있어야 함 |
